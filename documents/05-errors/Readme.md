## Errors

There are different kinds of errors that needs to be handled,
one of them is client side errors
400: bad input data
401: not authentication, not logged int
403: not authorized, not enough permission
404: not found, the resource the user is looking for is missing.

## Server:

It indicates a failure on the webserver while trying to process a request.

- app is failed
- time limit is exceeded

500: internal server error: means the server failed to process the request.
